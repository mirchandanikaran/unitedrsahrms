Employee Management System - Setup Guide

Quick setup for local development, backend + frontend, and basic verification.

HTML Guide

1) Prerequisites

  • Python 3.11+
  • Node.js 20+
  • PostgreSQL (recommended), or SQLite for local dev
For local development, SQLite works well and avoids PostgreSQL setup.

2) Clone and open project

git clone <your-repo-url>
cd HRMS

3) Backend setup

  1. Create virtual environment and install dependencies.
  2. Configure backend/.env.
  3. Seed sample data.
  4. Run FastAPI server.
cd backend
python -m venv venv

# Linux/macOS
source venv/bin/activate
# Windows (PowerShell)
# .\venv\Scripts\Activate.ps1

pip install -r requirements.txt

# Copy env file and edit values if needed
cp .env.example .env

# Seed demo data
python scripts/seed_data.py

# Run backend API
uvicorn app.main:app --reload --port 8000
If login returns HTTP 500 due to bcrypt/passlib mismatch, reinstall backend deps from requirements.txt so pinned versions apply.

4) Frontend setup

cd frontend
npm install
npm run dev

Open: http://localhost:3000

5) First-time onboarding (empty database)

  • Open http://localhost:3000 after backend/frontend are running.
  • You will be redirected to /setup.
  • Create your first admin account and complete initialization.
  • Then log in with the credentials you just created.

6) Verify setup

Backend

  • Health: http://localhost:8000/health
  • Swagger: http://localhost:8000/docs

Frontend

  • App: http://localhost:3000
  • Login should succeed with demo credentials

7) Run tests (optional but recommended)

Backend tests

cd backend
python -m pytest tests/ -v

Frontend tests + typecheck

cd frontend
npm run test:run
npx tsc --noEmit

8) Docker deployment (office server)

cp deployment/docker.env.example deployment/docker.env
# edit docker.env (passwords/secrets/origins)
docker compose up -d --build

On a fresh database, the app will open the first-time onboarding flow at /setup.

See DEPLOY_DOCKER.md for full server operations.
